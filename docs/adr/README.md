# Architecture Decision Records

Architecture Decision Records (ADRs) preserve significant decisions and the reasoning behind them.

## Lifecycle

An ADR may be Proposed, Accepted, Superseded, Deprecated or Rejected.

Accepted ADRs are not rewritten to change history. A later ADR may supersede them.

## Naming

```text
ADR-NNNN-short-descriptive-title.md
```

## Template

```markdown
# ADR-NNNN: Title

- Status: Proposed
- Date: YYYY-MM-DD
- Decision Owners: ...

## Context
## Decision
## Rationale
## Consequences
## Alternatives Considered
## Related Decisions and Evidence
```

The purpose is organisational memory, not administrative overhead.
