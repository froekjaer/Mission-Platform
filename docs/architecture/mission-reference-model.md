# Mission Reference Model

**Version:** 0.1  
**Status:** Foundation Draft

## Purpose

The Mission Reference Model (MRM) defines the core concepts used to describe mission-oriented reality.

It answers: **What exists?**

> The model describes reality—not software.

## Core concepts

- **Mission** — a meaningful purpose pursued in response to a real-world need, opportunity, obligation or condition.
- **Objective** — a defined outcome that contributes to a mission.
- **Capability** — the ability to produce an outcome or effect under specified conditions.
- **Service** — a defined means of delivering value or enabling a capability.
- **Actor** — a person, organisation, system or participant that performs actions, holds responsibilities or is affected by outcomes.
- **Asset** — something of value that must be used, protected, governed or observed.
- **Event** — a significant occurrence that may change state, trigger action or provide evidence.
- **State** — a condition of an entity at a particular time.
- **Workflow** — an organised progression of activities, decisions and state changes intended to produce an outcome.
- **Decision** — a selection among alternatives made by an authorised actor or mechanism.
- **Policy** — a rule, constraint or guiding statement that influences decisions and behaviour.
- **Resource** — something consumed, allocated or required to perform an activity or provide a capability.
- **Identity** — the persistent means by which an entity is distinguished and referenced over time.
- **Evidence** — an observation, measurement, record or artefact used to evaluate a claim, outcome, state or relationship.
- **Reality Anchor** — an explicit connection between a model element and an observable part of the real world.

## Initial relationship vocabulary

Examples include Mission **has Objective**, Objective **requires Capability**, Capability **is enabled by Service**, Actor **owns or performs**, Policy **constrains**, Event **changes State**, Workflow **uses Resource**, Evidence **supports or challenges**, and Reality Anchor **grounds** an element in reality.

Formal relationship rules are defined by the Mission Meta Model.
